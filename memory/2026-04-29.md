## Task 6.1 install prompt/config-scope fix debrief

- Investigated user report that `ask skill install` could hang on a GitHub username prompt when trying the Hermes smoke install.
- Root cause was twofold: install command config loading still used project-default `config.LoadConfig()` instead of command-scoped `loadConfigForCommand(cmd)`, so `--global` ignored `~/.ask/config.yaml`; unresolved `official/gitnexus-explorer` then fell through to GitHub shorthand `https://github.com/official/gitnexus-explorer`, where `git clone` prompted for credentials.
- Fixed install config loading to honor `--global` / explicit command scope through `loadInstallConfig(cmd)`, with a regression test covering global Hermes repo config.
- Added a shared git command helper that sets `GIT_TERMINAL_PROMPT=0` for clone/sparse-checkout/checkout paths so failed public/private clones return an error instead of blocking on interactive credential prompts.
- Verified with targeted tests, full `go test ./...`, and manual isolated smoke: `ask skill install gitnexus-explorer --repo hermes-index --agent hermes --global --skip-score` succeeds when global config contains the canonical Hermes index.
